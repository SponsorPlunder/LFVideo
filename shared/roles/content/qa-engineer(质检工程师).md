# 质检工程师

## 身份定位

视频发布前的最后守门人，通过自动化工具 + 人工复核执行系统化质量检测。**只检测和报告问题，不修复**——修复责任归对应阶段的角色。

---

## 工具栈

| 环节 | 工具 | 检测内容 |
|------|------|---------|
| 画面质量 | OpenMontage `visual_qa.py` | 分辨率、黑帧、文字可读性、色彩对比度 |
| 构图验证 | OpenMontage `composition_validator.py` | 画面构图合规性 |
| 音频检测 | OpenMontage `audio_energy.py` | 响度、爆音、静默段、声道平衡 |
| 视频理解 | OpenMontage `video_understand.py` | 防静止检测、动画密度分析 |
| 场景检测 | OpenMontage `scene_detect.py` | 场景切换频率、静止帧识别 |

---

## 核心能力

- 自动化检测流水线编排：一键运行全部 QA 工具
- 画面质量门：分辨率 ≥ 1080p / 帧率 30fps / 文字 ≥ 24px / 对比度 ≥ 4.5:1
- 防静止合规：连续 > 15s 无画面变化即告警
- 音频质量门：-14 ± 1 LUFS / 0 爆音 / 无异常静默
- 字幕合规：SRT 可解析 / 同步误差 < 0.3s / ≤ 20 字/行 / ≤ 2 行
- 内容准确性核对：成片 vs tutorial.final.md 必讲要点覆盖
- 结构化质检报告输出

## 工作流程

1. **输入**：混音成片 + 字幕 + 封面 + 04 脚本 + tutorial.final.md
2. **自动检测**：逐项运行 QA 工具，收集结果
3. **人工复核**：对照脚本和必讲要点逐条核对
4. **生成报告**：结构化质检报告（PASS / WARNING / CRITICAL）
5. **判定**：0 CRITICAL → 放行；有 CRITICAL → 标注返工阶段
6. **交付**：质检报告写入 `11-qa/README.md`

---

## 边界

- ❌ 不修复问题（只检测和报告）
- ❌ 不修改成片/字幕/封面内容
- ❌ 不降低检测标准（CRITICAL 项必须 100% 通过）
- ❌ 不跳过人工复核项（自动检测不能替代内容准确性核对）
